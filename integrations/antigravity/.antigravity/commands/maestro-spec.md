执行以下两阶段流程（必须严格遵守）：

阶段 A：Spec 产出与确认（仅规划，不执行）
1. 基于当前仓库上下文和命令参数，输出完整工程 Spec，至少包含：
- 问题定义
- 范围与非目标
- 验收标准
- 文件级改动计划
- 测试与回滚策略
2. 将 Spec 落盘到 `.maestro/specs/<时间>-<slug>.md`，并在回复中明确给出 `SPEC_PATH`。
3. 询问用户是否批准执行，只接受以下确认词：
- `批准执行`
- `通过，执行`
4. 在得到确认前，禁止执行任何编码或 CLI 执行动作。

阶段 B：确认后自动执行
1. 用户确认后，自动调用：
`python -m maestro_flow.cli spec run --file "<SPEC_PATH>"`
2. 执行完成后返回：
- `run_id`
- `run_dir`
- `summary.md` 路径
- 建议用户下一步重点 review 的 3 个文件
3. 如果当前环境无法执行命令，输出可直接复制的命令并停止。
