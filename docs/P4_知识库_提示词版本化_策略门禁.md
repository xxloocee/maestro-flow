# P4 实施说明：知识库注入、提示词版本化与策略门禁

P4 的目标是把多 Agent 协作从“可运行”提升到“可治理”：

- 让每个阶段基于项目知识库上下文执行
- 让每个阶段提示词具备可追踪版本与哈希
- 让输出在进入人工 Review 前先经过统一策略门禁

## 1. 知识库注入

实现位置：`src/maestro_flow/knowledge_base.py`

核心能力：
- 按 `knowledge.include_patterns` 收集仓库文件
- 去重并稳定排序，保证同仓库多次运行结果可复现
- 限制文件数、单文件截断长度、总字符预算
- 输出 `path + sha256 + snippet`，注入到阶段上下文 `context.knowledge`

运行产物：
- `knowledge_snapshot.json`

## 2. 提示词版本化

实现位置：`src/maestro_flow/prompting.py`

核心能力：
- 支持解析 prompt 文件 front matter：
  - `version`
  - `updated_at`（可选）
- 对提示词正文计算 `sha256`
- 生成阶段提示词注册表，写入：
  - `run_state.json -> prompts`
  - `prompt_registry.json`

推荐规范：

```md
---
version: v1.0.0
updated_at: 2026-03-24
---
这里是中文提示词正文
```

## 3. 策略门禁

实现位置：`src/maestro_flow/policy_gate.py`

默认规则：
- `TESTER_CASES_REQUIRED`
- `DEV_VERIFY_COMMAND_REQUIRED`
- `DEV_FILE_CHANGES_LIMIT`
- `REVIEWER_NO_CRITICAL`
- `SECURITY_SECRET_PATTERN_SCAN`

规则结果会写入：
- `policy_report.json`
- `run_state.json -> policies`

若存在阻断型失败：
- 运行状态置为 `failed`
- 错误码置为 `POLICY_GATE_FAILED`

## 4. 编排器接入点

实现位置：`src/maestro_flow/orchestrator.py`

关键接入：
1. 运行开始前收集知识库与提示词注册表。
2. 阶段执行时将 `knowledge + prompt_meta` 注入上下文。
3. DAG 全部完成后执行策略评估并落盘。
4. 阻断失败时中断后续流程，并走既有回滚策略。

## 5. 配置示例

`agents/agents.yaml`：

```yaml
knowledge:
  enabled: true
  include_patterns:
    - README.md
    - docs/**/*.md
    - agents/prompts/*.md
  max_files: 12
  max_chars_per_file: 2500
  max_total_chars: 12000

policy:
  enabled: true
  require_tester_cases: true
  require_dev_verification_commands: true
  max_dev_file_changes: 120
  block_on_critical_review: true
  secret_patterns:
    - AKIA[0-9A-Z]{16}
    - "(?i)api[_-]?key\\s*[:=]\\s*['\\\"][^'\\\"]+['\\\"]"
    - "(?i)password\\s*[:=]\\s*['\\\"][^'\\\"]+['\\\"]"
```
