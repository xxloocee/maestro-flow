# P6 实施说明：执行闭环（自动改代码 + 自动测试 + 自动修复）

P6 的目标是把多 Agent 输出从“建议”推进到“可执行”。

核心路径：
1. 应用开发阶段给出的文件改动。
2. 执行验证命令并收集结果。
3. 若失败，则调用 Debugger 生成修复改动。
4. 进入下一轮，直到通过或达到最大修复轮次。
5. 闭环成功后自动刷新 reviewer 结论。

## 1. 新增配置

位置：`agents/agents.yaml` -> `workflow`

```yaml
workflow:
  execution_enabled: false
  max_fix_loops: 2
  command_timeout_seconds: 180
  execution_workspace_mode: inplace
  cleanup_execution_workspace: false
  sync_back_on_success: false
  sync_back_only_if_policies_pass: true
  sync_back_conflict_action: block
  allowed_execution_commands:
    - python -m pytest
    - pytest
    - python -m unittest
    - npm test
    - npm run test
    - pnpm test
    - pnpm run test
    - yarn test
    - yarn run test
  command_policies: []
  unmatched_command_action: block
  blocked_command_fragments:
    - rm -rf
    - del /f /s /q
    - "format "
    - shutdown
    - reboot
    - git reset --hard
    - git checkout --
```

含义：
- `execution_enabled`：是否开启执行闭环。
- `max_fix_loops`：最大修复轮次。
- `command_timeout_seconds`：单条命令超时。
- `execution_workspace_mode`：执行空间模式（`inplace` 或 `copy`）。
- `cleanup_execution_workspace`：是否在运行后删除隔离副本。
- `sync_back_on_success`：`copy` 模式成功后是否回写改动到主工作区。
- `sync_back_only_if_policies_pass`：仅在策略门禁通过时回写。
- `sync_back_conflict_action`：回写冲突策略（`block` 默认阻断，`overwrite` 允许覆盖并记录冲突）。
- `allowed_execution_commands`：命令白名单前缀（兼容旧配置）。
- `command_policies`：规则化命令策略（`prefix/regex` + 参数要求 + 风险动作，优先于前缀白名单）。
- `unmatched_command_action`：命令未命中策略时的处理方式（`block` 或 `warn`）。
- `blocked_command_fragments`：命中即拒绝执行的危险命令片段。

`command_policies` 示例：

```yaml
workflow:
  command_policies:
    - mode: regex
      pattern: "^python\\s+-m\\s+pytest\\b"
      description: pytest 命令策略
      required_args:
        - "-q"
      missing_required_action: warn
      forbidden_args:
        - "--maxfail=0"
      forbidden_arg_action: block
  unmatched_command_action: block
```

## 2. 数据结构扩展

位置：`src/maestro_flow/contracts.py`

- `FileChange` 新增 `content` 字段（用于写入完整文件内容）。
- `DebugOutput` 新增：
  - `file_changes`
  - `fix_commands`

这样 Debugger 可以直接返回可执行修复动作。

## 3. 执行器模块

位置：`src/maestro_flow/executor.py`

提供能力：
1. `apply_file_changes`：
- 应用 create/update/delete
- 拒绝路径越界写入
- 生成统一 diff patch（`fix_round_<n>.patch`）

2. `run_commands`：
- 命令放行策略：`command_policies` -> `allowed_execution_commands`
- 危险命令片段前置拦截：`blocked_command_fragments`
- 记录 stdout/stderr/exit_code
- 超时返回 124
- 风险动作：命中违规时可配置 `warn` 或 `block`

## 4. 编排器接入

位置：`src/maestro_flow/orchestrator.py`

新增逻辑：
- 在 DAG 阶段后，若 `execution_enabled=true`，执行 P6 闭环。
- 支持 `execution_workspace_mode=copy`：在隔离副本中执行，不污染主工作区。
- 支持 `sync_back_on_success=true`：隔离副本执行成功后可回写主工作区。
- 闭环失败时：
  - `manifest.status = failed`
  - `error_code = EXECUTION_LOOP_FAILED`
- 闭环成功后自动二次调用 reviewer，并覆盖最新 `stage_reviewer.json`。

新增产物：
- `file_apply_round_<n>.json`
- `test_report_round_<n>.json`
- `fix_round_<n>.md`
- `fix_round_<n>.patch`
- `stage_debugger_loop_<n>.json`
- `stage_reviewer_post_execution.json`
- `sync_back_baseline.json`
- `sync_back_report.json`
- `policy_report.json` 中追加执行闭环规则：
  - `EXECUTION_COMMAND_ALLOWED`
  - `EXECUTION_COMMAND_TIMEOUT`
  - `EXECUTION_COMMAND_WARNINGS`

说明：即使执行闭环失败，仍会写出上述策略结果，便于在 PR 评论中快速定位风险与阻断点。
默认仅在策略门禁通过时回写；若设置 `sync_back_only_if_policies_pass=false`，则可在门禁失败时仍尝试回写。
`copy` 模式下会在执行开始时生成回写基线（`sync_back_baseline.json`）；回写时若发现主工作区同路径文件已变化，会按 `sync_back_conflict_action` 处理。

## 5. CLI 运行时开关

位置：`src/maestro_flow/cli.py`

新增参数：
- `run --execution-loop`
- `spec run --execution-loop`
- `run --execution-isolated`
- `spec run --execution-isolated`
- `run --execution-sync-back`
- `spec run --execution-sync-back`

作用：临时开启闭环，不必修改 `agents/agents.yaml`；`--execution-isolated` 会强制使用 `copy` 隔离模式，`--execution-sync-back` 会在成功后回写主工作区。

当 `sync_back_conflict_action=block` 导致回写失败时，可使用人工决策重试：
- `sync-back plan --run-id <run_id>`：生成 `sync_back_decisions.json` 决策清单。
- `sync-back apply --run-id <run_id>`：读取清单并执行回写。

决策项支持：
- `decision=apply`：按隔离执行结果覆盖主工作区。
- `decision=keep_local`：保留主工作区当前版本。

执行后会生成 `sync_back_manual_report.json`，并把摘要写入 `run_state.json -> execution.sync_back_manual`。

## 6. 测试覆盖

新增测试：
- `tests/test_executor.py`
  - 文件应用与 patch 产物
  - 非白名单命令阻断
  - regex 策略放行
  - 危险命令片段阻断
- `tests/test_execution_loop.py`
  - 第一轮通过
  - 依赖 Debugger 第二轮修复通过
  - 闭环成功后二次 reviewer 刷新
  - copy 隔离模式不污染主工作区
  - copy 隔离模式成功回写主工作区
  - copy 隔离模式回写冲突默认阻断 / 可配置覆盖
- `tests/test_sync_back_ops.py`
  - 回写决策清单生成（冲突检测）
  - 手工回写决策执行（`keep_local` / `apply`）

## 7. 已知限制

1. 当前文件改动模型是“整文件 content 写入”，还未支持 AST 级或 hunk 级精细补丁。
2. 命令策略当前是 token/正则层匹配，尚未做 shell AST 级安全审计。
3. 闭环仅刷新 reviewer，不会自动重跑架构/测试策略阶段。
